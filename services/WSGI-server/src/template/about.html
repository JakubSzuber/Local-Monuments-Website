Note:

This application's purpose is not to be a ready-to-use working application to use on the Internet, it's rather the example core Infrastructure of how the containerized website should look like, so this app doesn't contain a few required security features like e.g. TLS if this app would be created to be on the Internet.

Normally, if this application was designed to be hosted on your local computer for global use, then a user would send the HTTPS request to the basic Nginx or Ingress-NGINX and then wait for a response.

1. Initialization of services

In order to set up the application, each container is launched and the necessary configurations are made, including the connection between the Postgres database and the WSGI Gunicorn server. Additionally, SQL queries are run to populate the database with initial data that will be used to display content on the Home and About pages

2. Routing the HTTP traffic by Nginx

To manage the incoming HTTP traffic, Nginx acts as a web-reverse-proxy server, routing traffic to the application server and awaiting a response

3. Taking the traffic by Gunicorn

Gunicorn, serving as a Web Server Gateway Interface (WSGI) application server, receives the traffic from Nginx and handles requests accordingly. If the request is for static content, Gunicorn simply sends the content back to Nginx, which then delivers it to the user

4. Generating dynamic HTTP Content with Gunicorn and Flask

However, if the request requires backend logic, Gunicorn works with the Flask framework to execute necessary SQL queries, select data from the Postgres database, and create dynamic website content that is returned to Nginx as the HTTP content
